Deep-dive - ML

What I learned building an emotion recognition pipeline

Key idea: lock down consistent keys before merging multiple feature tables. Inspect columns with head(), dtypes, and value_counts() to catch drift early.

# pandas merge skeleton
import pandas as pd

df_gaze = pd.read_csv("gaze.csv")
df_head = pd.read_csv("head.csv")
df_labels = pd.read_csv("labels.csv")

ensure common key exists and matches dtype
for d in (df_gaze, df_head, df_labels):
d["clip_id"] = d["clip_id"].astype(str)

features = df_gaze.merge(df_head, on="clip_id", how="inner")
data = features.merge(df_labels, on="clip_id", how="inner")
data.to_csv("dataset_final.csv", index=False)

Common pitfalls: mismatched dtypes for keys, silent suffix collisions, and accidental outer joins that produce NaNs in labels. Keep merges inner until coverage is verified, then widen if needed.

Guide - Web

Shipping a one-file blog on GitHub Pages

  1. Create a repo (any name) and add index.html with this file’s content.
  2. Commit and push, or upload via the web UI.
  3. Repo → Settings → Pages → Set Branch: main, Folder: / (root) → Save.
  4. Open the URL shown at the top of the Pages section — that’s your blog.

Optional: pick a custom theme later with Jekyll, but this file is already production-ready and fast.

Speeding up CSV feature engineering

  • Specify dtypes on read_csv to save RAM.
  • Use category for low-cardinality strings (labels).
  • Validate merges by counting rows before/after each step.
  • Write outputs with index=False and newline handling.
